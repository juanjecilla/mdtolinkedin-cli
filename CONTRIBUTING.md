# Contributing

Thanks for contributing to `mdtolinkedin`.

## Prerequisites

- Rust stable toolchain (`rustup default stable`)
- Git

## Local Setup

```bash
git clone https://github.com/juanjecilla/mdtolinkedin-cli.git
cd mdtolinkedin-cli
cargo build
```

## Development Workflow

1. Create a branch with the `feature/` prefix.
2. Keep changes focused and add tests when behavior changes.
3. Run the standard checks before opening a PR:

```bash
cargo fmt -- --check
cargo clippy -- -D warnings
cargo test
```

If you have `make` available, the same checks are available via:

```bash
make ci
```

## Tests

- Unit tests live next to code under `src/`.
- Integration tests live in `tests/`.
- Fixture updates are generated by `scripts/update_fixtures.py`.
